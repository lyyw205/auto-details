# /generate-figma-make-prompt — Figma Make 프롬프트 생성

## Purpose
v2 HTML 상세페이지(`output/{product}-detail.html`)를 기반으로,
**Figma Make에 바로 입력할 수 있는 최종 프롬프트**를 생성합니다.

HTML의 구조·콘텐츠·시각 효과를 자연어 디자인 지시문으로 변환하고,
각 이미지 영역에 AI 이미지 생성 프롬프트를 포함합니다.

## Context
- 입력 HTML: `output/{product}-detail.html` (v2 generate-html 결과물)
- 또는 레이아웃 JSON: `output/{product}-layout.json` (v1도 가능)
- 섹션 분류 체계: `skills/section-taxonomy.json`
- 제품 이미지가 함께 제공된다는 전제

## Input
1. **HTML 파일**: `output/{product}-detail.html`
2. **제품 정보** (HTML에서 추출 가능하지만 보충 가능):
   - 제품명 (한글 + 영문)
   - 브랜드명
   - 브랜드 컬러 hex
   - 제품 카테고리 (가전, 뷰티, 식품 등)
3. **제품 이미지**: 유저가 함께 첨부할 실제 제품 사진 (Figma Make에 같이 입력)
4. (선택) **디자인 톤 선호**: 프리미엄/미니멀/내추럴/테크 등

## Processing

### 1단계: HTML 파싱 — 구조 추출

HTML에서 다음을 추출합니다:
- 섹션 수, 각 섹션의 id와 composition 타입
- 각 섹션의 텍스트 콘텐츠 (카피라이팅)
- 이미지 플레이스홀더의 `data-ai-prompt`, `data-ai-style`, `img-label`
- 브랜드 컬러 (`--brand-main`, `--accent`)
- 배경 패턴 (gradient 교차)

### 2단계: 프롬프트 구조 작성

최종 프롬프트는 아래 구조를 따릅니다:

```
# [제품명] 상세페이지 디자인

## 디자인 개요
(전체 톤, 스타일, 분위기, 타겟)

## 디자인 시스템
(컬러, 타이포그래피, 시각 효과 지침)

## 섹션별 디자인 지시
(각 섹션의 레이아웃 + 콘텐츠 + 이미지 가이드)

## 이미지 가이드
(각 이미지 영역의 AI 프롬프트 + 설명)
```

### 3단계: 섹션별 프롬프트 변환

각 HTML 섹션을 자연어 디자인 지시문으로 변환합니다:

#### 변환 규칙

| HTML 요소 | Figma Make 프롬프트 표현 |
|-----------|------------------------|
| `composed-section` | "배경 이미지 위에 텍스트를 오버레이하는 히어로 레이아웃" |
| `grid grid-cols-2` | "좌우 2분할 레이아웃 (비율 1:1)" |
| `flex flex-col` | "수직 나열, 가운데 정렬" |
| `glass-card` | "글래스모피즘 카드 (반투명 배경 + 블러)" |
| `text-gradient-brand` | "브랜드 컬러 그라데이션 텍스트" |
| `bg-dark-gradient-1` | "진한 다크 네이비 그라데이션 배경" |
| `bg-light-gradient-1` | "밝은 연회색 그라데이션 배경" |
| `num-badge` | "번호 배지 (둥근 사각형, 브랜드 컬러)" |
| `check-item` | "체크 아이콘 + 텍스트 리스트" |
| `rounded-2xl` | "큰 라운드 모서리" |
| `text-shadow-hero` | "텍스트에 드롭 섀도우 (가독성 확보)" |
| `divider-gradient` | "미묘한 그라데이션 구분선" |

#### 이미지 영역 변환

```
📸 이미지 영역: [img-label 한글 설명]
   위치: [섹션 내 위치 설명]
   크기: [너비 × 높이]
   스타일: [data-ai-style]
   AI 이미지 프롬프트: "[data-ai-prompt]"
```

### 4단계: 디자인 품질 지침 추가

프롬프트 말미에 아래 품질 지침을 항상 포함합니다:

```
## 디자인 품질 지침
- 860px 고정 너비 (모바일 상세페이지 기준)
- 모든 섹션 배경에 subtle gradient 적용 (단색 금지)
- 카드 요소에 글래스모피즘 (반투명 + backdrop-blur)
- 텍스트-이미지 오버레이 시 반드시 그라데이션 오버레이 + 텍스트 섀도우
- 인접 섹션 배경 교차 (같은 톤 연속 금지)
- 타이포그래피: 한국어 Pretendard 폰트, tracking-tight
- 이미지 플레이스홀더: 실제 이미지가 들어갈 영역 표시
```

## Output
- 파일: `output/{product}-figma-make-prompt.md`
- 형식: Markdown (Figma Make에 복사-붙여넣기 가능)

## 프롬프트 길이 가이드
- **디자인 개요**: 3~5줄
- **디자인 시스템**: 10~15줄
- **섹션별 지시**: 섹션당 5~10줄 (총 분량의 70%)
- **이미지 가이드**: 이미지당 3~4줄
- **품질 지침**: 5~8줄
- **전체 길이**: 1,500~3,000자 (한국어 기준)

## 주의사항
- Figma Make는 자연어 지시를 이해하므로 CSS 클래스명을 그대로 쓰지 않음
- 디자인 의도를 "결과 지향적"으로 기술 (방법이 아닌 결과물 묘사)
- 이미지 영역은 실제 제품 이미지가 들어가므로 "이 위치에 제품 이미지 배치" 형태로 기술
- 한국어로 작성 (이미지 프롬프트만 영문)
